I'm not interested in getting on a high horse about the technologies I use; there's enough coding zeal on the Interwebs. For my blog's primary audience, an excruciating list of web tech is blas√©. But I will include this section for the sake of the curious, perhaps for the newcomer, how I cobbled together this site. While I've put my 10,000 hours into programming, I take a look back bewildered: programming is still something new every day, which is both thrilling and frustrating, "Why isn't this easier?" For the uninitiated, an unintuitive jigsaw of lines of text jammed together. So how does anyone ever get to the point of realizing their own website? I don't have a great answer. A colophon is the least I can do.

And there's probably smarter ways to do it. The work I did for this site, does it offer building blocks for future sites? Is it underengineered? Over? Was I lazy? Or was I not lazy enough?

The source is pretty traditional HTML5, CSS3, and JavaScript, written by hand on a MacBook using MacVim, a lot of prodding in Chrome DevTools, and auto-refreshing the page via LiveReload. I navigate these via a command-line interface when I can, via iTerm2.

For a site this small and content unchanging, there is no CMS. No database. Nothing dynamic. The content is static, versioned in Git, generated into a site by Jekyll. It is hosted by GitHub Pages. The source code is available on GitHub as well.

This site's template is a modification of Jekyll's stock ___. The preferred font is the humanist sans-serif Optima, widely installed on Mac OS X. On Windows, this site will degrade to either humanist Segoe or Helvetica. Pygments highlights code snippets in a Solarized color scheme.

For some of the layouts, behaviors, and animations that are tricky in CSS, I use the JavaScript libraries AngularJS, for more maintainable JavaScript components & organization, and jQuery, for lower-level DOM manipulation.
